## ü™£ **STEP 1: Create AWS S3 Bucket**

### **1.1 Login to AWS Console**
1. Go to [AWS Console](https://console.aws.amazon.com/)
2. Sign in with your AWS account
3. Search for "S3" in the services

### **1.2 Create New Bucket**
1. Click **"Create bucket"**
2. Fill in the following details:

#### **Basic Configuration:**
```
Bucket name: educational-books-storage-[your-name]
             (must be globally unique)

Region: us-east-1 (or closest to your users)

Bucket Versioning: Enable (recommended for file recovery)
```

#### **Public Access Settings:**
```
‚ùå UNCHECK "Block all public access"

Specifically uncheck:
- ‚ùå Block public access to buckets and objects granted through new access control lists (ACLs)
- ‚ùå Block public access to buckets and objects granted through any access control lists (ACLs)

Keep checked:
- ‚úÖ Block public access to buckets and objects granted through new public bucket or access point policies
- ‚úÖ Block public and cross-account access to buckets and objects through any public bucket or access point policies
```

#### **Bucket Encryption:**
```
Default encryption: Enable
Encryption type: Server-side encryption with Amazon S3 managed keys (SSE-S3)
```

### **1.3 Configure Bucket Policy**
After creating the bucket, add this policy for public read access to cover images:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadCoverImages",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": [
                "arn:aws:s3:::YOUR-BUCKET-NAME/books/covers/*",
                "arn:aws:s3:::YOUR-BUCKET-NAME/ads/images/*"
            ]
        }
    ]
}
```

**Replace `YOUR-BUCKET-NAME` with your actual bucket name**

### **1.4 Configure CORS Policy**
Add CORS configuration for web application access:

```json
[
    {
        "AllowedHeaders": ["*"],
        "AllowedMethods": ["GET", "PUT", "POST", "DELETE", "HEAD"],
        "AllowedOrigins": [
            "http://localhost:3000",
            "http://localhost:5173",
            "https://yourdomain.com"
        ],
        "ExposeHeaders": ["ETag"],
        "MaxAgeSeconds": 3000
    }
]
```

## üë§ **STEP 2: Create IAM User and Permissions**

### **2.1 Create IAM User**
1. Go to **IAM** service in AWS Console
2. Click **Users** ‚Üí **Create user**
3. Configure:
   ```
   User name: educational-book-app-user
   Access type: ‚úÖ Programmatic access
   ```

### **2.2 Create Custom IAM Policy**
1. In IAM, go to **Policies** ‚Üí **Create policy**
2. Choose **JSON** tab
3. Paste this policy:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "S3BucketPermissions",
            "Effect": "Allow",
            "Action": [
                "s3:ListBucket",
                "s3:GetBucketLocation",
                "s3:GetBucketVersioning"
            ],
            "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME"
        },
        {
            "Sid": "S3ObjectPermissions",
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:PutObject",
                "s3:DeleteObject",
                "s3:PutObjectAcl",
                "s3:GetObjectAcl"
            ],
            "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
        }
    ]
}
```

4. Name the policy: `EducationalBookAppS3Policy`
5. Save the policy

### **2.3 Attach Policy to User**
1. Go back to your user
2. Click **Add permissions** ‚Üí **Attach existing policies directly**
3. Search and select: `EducationalBookAppS3Policy`
4. Click **Add permissions**

### **2.4 Generate Access Keys**
1. Go to your user's **Security credentials** tab
2. Click **Create access key**
3. Select **Application running outside AWS**
4. **IMPORTANT**: Copy and save:
   - ‚úÖ Access Key ID
   - ‚úÖ Secret Access Key
   - ‚ö†Ô∏è You won't be able to see the secret key again!

### **3.3 Environment Variables Breakdown**

| Variable | Description | Required | Example |
|----------|-------------|----------|---------|
| `AWS_ACCESS_KEY_ID` | Your IAM user's access key | ‚úÖ Yes | `AKIA1234567890EXAMPLE` |
| `AWS_SECRET_ACCESS_KEY` | Your IAM user's secret key | ‚úÖ Yes | `wJalrX...` |
| `AWS_REGION` | AWS region where bucket exists | ‚úÖ Yes | `us-east-1` |
| `AWS_S3_BUCKET` | Your S3 bucket name | ‚úÖ Yes | `educational-books-storage` |
| `AWS_S3_ENDPOINT` | Custom endpoint (LocalStack/MinIO) | ‚ùå No | `http://localhost:4566` |
| `AWS_S3_FORCE_PATH_STYLE` | Path-style URLs (for LocalStack) | ‚ùå No | `true` |

## üîí **STEP 6: Security Best Practices**

### **6.1 Access Key Security**
- ‚ùå Never commit access keys to code repository
- ‚úÖ Use environment variables only
- ‚úÖ Rotate access keys regularly (every 90 days)
- ‚úÖ Use IAM roles in production (instead of access keys)

### **6.2 Bucket Security**
- ‚úÖ Only make cover images publicly readable
- ‚ùå Never make PDF files public
- ‚úÖ Use signed URLs for PDF access
- ‚úÖ Enable bucket logging for audit trails

### **6.3 Network Security**
- ‚úÖ Use HTTPS only for all S3 requests
- ‚úÖ Configure proper CORS policies
- ‚úÖ Restrict access by IP if possible

## üêõ **Common Issues and Troubleshooting**

### **Issue 1: Access Denied Error**
```
Error: Access Denied
```
**Solutions**:
- ‚úÖ Check IAM policy permissions
- ‚úÖ Verify bucket name in policy matches actual bucket
- ‚úÖ Ensure access keys are correct
- ‚úÖ Check bucket policy doesn't conflict

### **Issue 2: Bucket Not Found**
```
Error: The specified bucket does not exist
```
**Solutions**:
- ‚úÖ Verify bucket name in .env file
- ‚úÖ Check AWS region matches bucket region
- ‚úÖ Ensure bucket was created successfully

### **Issue 3: CORS Error**
```
Error: CORS policy blocks request
```
**Solutions**:
- ‚úÖ Add your domain to CORS allowed origins
- ‚úÖ Include necessary headers in CORS config
- ‚úÖ Check request method is allowed

### **Issue 4: File Too Large**
```
Error: File size exceeds limit
```
**Solutions**:
- ‚úÖ Check multer configuration in your code
- ‚úÖ Verify AWS S3 multipart upload settings
- ‚úÖ Increase limits if necessary

### **Issue 5: Signed URL Issues**
```
Error: SignedURL expired or invalid
```
**Solutions**:
- ‚úÖ Check AWS credentials are valid
- ‚úÖ Verify system clock is synchronized
- ‚úÖ Ensure correct bucket and key in URL generation
